# Nova Serious Workflow

If the full pack is available, use `workflows/nova-serious-workflow.md` as the canonical public workflow. If this skill folder was copied alone, the compact rules below are sufficient to operate safely.

For Codex:

1. Read the project entry files before editing.
2. Prefer the user's current request and real files over stored notes.
3. Keep task IDs explicit, such as `T001`.
4. Use Markdown checkboxes in task boards and checklists.
5. Archive before compacting or restructuring important notes.
6. Stop before risky actions listed in the workflow.
7. Verify before marking complete.

Final report:

```text
completed:
files changed:
task updated:
intelligence used:
verification:
not tested:
risks:
next:
```
