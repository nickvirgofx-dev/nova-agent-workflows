# Nova Serious Workflow

If the full pack is available, use `workflows/nova-serious-workflow.md` as the canonical public workflow. If this skill folder was copied alone, the compact rules below are sufficient to operate safely.

For Codex:

1. Read the project entry files before editing.
2. Prefer the user's current request and real files over stored notes.
3. Use the six-layer stack when the work is substantial: context, ingestion/knowledge, planning/spec, task board/checklist, verification/risk gate, delivery/sync-back.
4. Keep task IDs explicit, such as `T001`.
5. Use Markdown checkboxes in task boards and checklists.
6. Archive before compacting or restructuring important notes.
7. Stop before risky actions listed in the workflow.
8. Verify before marking complete.
9. Sync results, decisions, lessons, bug patterns, and next tasks back into project notes.

Final report:

```text
completed:
files changed:
task updated:
intelligence used:
verification:
not tested:
risks:
next:
```
